## Metodologia

O projeto seguiu uma abordagem **Scrum simplificada** com uma única sprint de
2 semanas. O backlog foi construído a partir dos requisitos da atividade A2,
mapeando cada funcionalidade em **Épicos → Features → User Stories → Subtasks**.

Cada user story recebeu story points (escala Fibonacci) e critérios de
aceitação testáveis. O quadro Kanban foi utilizado para acompanhar o progresso
diário das tarefas.
